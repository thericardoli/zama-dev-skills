# forge-fhevm Testing Overview

This directory explains how to bring the contract development patterns from `zama-fhevm-solidity-core` into Foundry/forge-fhevm tests. Read this file first to identify the test category, then open the matching topic guide.

## Test Categories to Write First

An FHEVM contract should cover at least these paths:

| Test objective | File to read | What it proves |
| --- | --- | --- |
| Encrypted input reaches the contract successfully | `encrypt.md` | Handles/proofs generated by `encrypt*` can be consumed by `FHE.fromExternal` |
| Encrypted computation results are correct | `decrypt.md` | FHE operations produce the expected result |
| Users can actually decrypt their own values | `decrypt.md` + `acl.md` | Both `FHE.allowThis` and `FHE.allow(user)` are correct |
| Unauthorized users cannot decrypt | `acl.md` | ACL has not been over-granted |
| Public decrypt flow is correct | `decrypt.md` | Public flag, KMS proof, and `FHE.checkSignatures` encoding line up |
| ERC7984 token behavior | `erc7984.md` | Mint/burn/transfer/operator/event amount/disclose paths are correct |
| Boundary values and failure paths | `fuzz-and-errors.md` | Overflow/underflow, wrong target, wrong signer, and wrong ACL are covered |

Tests that only assert with `decrypt(...)` are fast, but they do not prove ACL or real user-decrypt flows.

## Minimal Test Skeleton

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.27;

import {FhevmTest} from "forge-fhevm/FhevmTest.sol";
import {FHE, euint64, externalEuint64} from "@fhevm/solidity/lib/FHE.sol";
import {ZamaEthereumConfig} from "@fhevm/solidity/config/ZamaConfig.sol";

contract ConfidentialVault is ZamaEthereumConfig {
    mapping(address => euint64) private _balances;

    function deposit(externalEuint64 encryptedAmount, bytes calldata proof) external {
        euint64 amount = FHE.fromExternal(encryptedAmount, proof);
        euint64 next = FHE.add(_balances[msg.sender], amount);

        _balances[msg.sender] = next;
        FHE.allowThis(next);
        FHE.allow(next, msg.sender);
    }

    function balanceOf(address account) external view returns (euint64) {
        return _balances[account];
    }
}

contract ConfidentialVaultTest is FhevmTest {
    ConfidentialVault vault;

    function setUp() public override {
        super.setUp();
        vault = new ConfidentialVault();
    }

    function test_deposit() public {
        (externalEuint64 amount, bytes memory proof) = encryptUint64(100, address(vault));
        vault.deposit(amount, proof);
        assertEq(decrypt(vault.balanceOf(address(this))), 100);
    }
}
```

Key points:

- The test contract inherits `FhevmTest`.
- `setUp()` calls `super.setUp()` first when overridden.
- The contract under test inherits `ZamaEthereumConfig`.
- The contract receives user encrypted input through `FHE.fromExternal`.
- ACL is set again after storing a new handle.

## File Guide

- `encrypt.md`: how to use `encryptBool`, `encryptUintXX`, and `encryptAddress` to generate test inputs.
- `decrypt.md`: how to choose between `decrypt`, `publicDecrypt`, `buildDecryptionProof`, and `userDecrypt`.
- `acl.md`: how to test `allowThis`, `allow`, `allowTransient`, the public-decrypt flag, and permission propagation.
- `erc7984.md`: how to test OpenZeppelin ERC7984 confidential tokens.
- `fuzz-and-errors.md`: how to write fuzz tests, boundary tests, wrapping checks, and common failure-path coverage.
- API signatures and helper details: see `../api/README.md`.
