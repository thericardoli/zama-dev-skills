# 开发模式：ERC7984 confidential token 集成

## ERC7984 是什么

ERC7984 是 OpenZeppelin confidential contracts 中的 confidential fungible token 基础实现，用 FHEVM encrypted balances 和 encrypted transfer amount 实现隐私 token。它适合需要隐藏余额和转账金额，同时保留 ERC 类 token 工作流的项目。

官方仓库：

- `https://github.com/OpenZeppelin/openzeppelin-confidential-contracts`
- Zama 示例通常在 `https://github.com/zama-ai/fhevm` 的 docs/examples/openzeppelin 下。

## 基本依赖

合约通常需要：

```solidity
import {Ownable2Step, Ownable} from "@openzeppelin/contracts/access/Ownable2Step.sol";
import {ERC7984} from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";
import {FHE, euint64, externalEuint64} from "@fhevm/solidity/lib/FHE.sol";
import {ZamaEthereumConfig} from "@fhevm/solidity/config/ZamaConfig.sol";
```

具体 import 路径以当前 OpenZeppelin confidential contracts 版本为准。

## 最小 token 示例

```solidity
// SPDX-License-Identifier: BSD-3-Clause-Clear
pragma solidity ^0.8.27;

import {Ownable2Step, Ownable} from "@openzeppelin/contracts/access/Ownable2Step.sol";
import {ERC7984} from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";
import {FHE, euint64} from "@fhevm/solidity/lib/FHE.sol";
import {ZamaEthereumConfig} from "@fhevm/solidity/config/ZamaConfig.sol";

contract ConfidentialToken is ZamaEthereumConfig, ERC7984, Ownable2Step {
    constructor(
        address owner,
        uint64 initialAmount,
        string memory name_,
        string memory symbol_,
        string memory contractURI_
    ) ERC7984(name_, symbol_, contractURI_) Ownable(owner) {
        euint64 encryptedAmount = FHE.asEuint64(initialAmount);
        _mint(owner, encryptedAmount);
    }
}
```

该示例使用明文 `initialAmount` 初始化供应量，适合公开 tokenomics。若初始供应量也需要保密，改用 confidential mint。

## Visible mint 和 confidential mint

Visible mint：金额公开，适合公开发行计划。

```solidity
function mintVisible(address to, uint64 amount) external onlyOwner {
    euint64 encryptedAmount = FHE.asEuint64(amount);
    _mint(to, encryptedAmount);
}
```

Confidential mint：金额由 minter 加密，适合隐藏发行量或分配金额。

```solidity
function mintConfidential(
    address to,
    externalEuint64 encryptedAmount,
    bytes calldata inputProof
) external onlyOwner {
    euint64 amount = FHE.fromExternal(encryptedAmount, inputProof);
    _mint(to, amount);
}
```

具体 `_mint` 签名以当前 OpenZeppelin confidential contracts 版本为准。

## 转账接口

ERC7984 confidential transfer 通常接收 recipient、encrypted amount handle 和 input proof。ABI 中 `externalEuint64` 有时会显示为 `bytes32`，但语义上仍是经过 proof 验证的 external encrypted input。

```solidity
function transferWithEncryptedAmount(
    address recipient,
    externalEuint64 encryptedAmount,
    bytes calldata inputProof
) external {
    euint64 amount = FHE.fromExternal(encryptedAmount, inputProof);
    FHE.allowTransient(amount, address(confidentialToken));
    confidentialToken.confidentialTransfer(recipient, amount);
}
```

## Cross-contract 调用

如果当前合约持有某个 `euint64 amount`，并要把它交给 ERC7984 token 合约执行转账，需要临时授权 token：

```solidity
FHE.allowTransient(amount, address(confidentialToken));
confidentialToken.confidentialTransfer(recipient, amount);
```

auction、vesting、AMM、wrapper 都会用到这个模式。

## 余额读取

```ts
const balanceHandle = await token.confidentialBalanceOf(user.address);
```

是否能 user decrypt 取决于 ERC7984 实现和当前用户 ACL。测试不能只断言 handle 存在，关键路径要覆盖 user decrypt 成功与未授权失败。

## Vesting 模式

Confidential vesting wallet 通常把时间条件保持为明文，把金额保持为 encrypted：

```solidity
function releasable(address token) public returns (euint64) {
    euint128 vested = vestedAmount(token, uint48(block.timestamp));
    euint128 releasedAmount = released(token);
    ebool ok = FHE.ge(vested, releasedAmount);
    return FHE.select(ok, FHE.asEuint64(FHE.sub(vested, releasedAmount)), FHE.asEuint64(0));
}
```

释放时把 releasable amount transient 授权给 token：

```solidity
euint64 amount = releasable(token);
FHE.allowTransient(amount, token);
euint64 sent = IERC7984(token).confidentialTransfer(owner(), amount);
```

## 集成注意事项

- ERC7984 通常使用 `euint64` 作为金额宽度；不要未经确认改成别的宽度。
- 初始 mint 用明文 amount 转 encrypted amount 会暴露初始供应量；如果业务要求完全隐私，需要设计 encrypted mint。
- `confidentialTransfer` 的 amount 必须由 sender 加密并由 sender 发交易。
- 检查 zero address、zero balance、insufficient balance 等错误路径。
- 和普通 ERC20 wrapper、DEX、vesting、bridge 集成时，明确哪些金额可以公开，哪些必须保密。
- cross-contract 使用 encrypted amount 前，检查是否需要 `allowTransient`。
- wrapper/unwrap 会把一部分信息公开到 ERC20 世界，先写清隐私假设。

## 常见扩展

- owner-only mint：如果 mint amount 是公开管理操作，可用 `FHE.asEuint64`；如果 mint amount 也保密，应改为 encrypted input。
- burn：同 transfer 一样要验证 encrypted amount，并处理余额不足。
- wrapper：ERC20 与 ERC7984 互转时，wrap/unwrap 的公开金额会影响隐私假设。
- compliance：可通过 user decrypt delegation 或 selective disclosure 设计合规读取，但必须显式授权和记录。
- sealed-bid auction：用 ERC7984 收取 encrypted bid，`FHE.select` 更新最高价和 encrypted winner，结束后 public decrypt winner。
