# 测试 ERC7984 confidential token

本文件只讲 Hardhat/`@fhevm/hardhat-plugin` 怎么测 ERC7984 行为。具体 token、mint、burn、wrapper、disclose 接口以项目实现为准。

## 测试面

| 行为 | Hardhat FHEVM API | 重点 |
| --- | --- | --- |
| balance decrypt | `userDecryptEuint(FhevmType.euint64, ...)` | owner 能解密，第三方不能 |
| mint / transfer input | `createEncryptedInput(...).add64(...).encrypt()` | target/user 绑定正确 |
| operator / transferFrom | encrypted input 绑定 operator | 未授权失败，授权成功，过期失败 |
| existing handle transfer | user decrypt + ACL | 不能拿别人的 handle 花钱 |
| event amount | logs + user decrypt | sender/recipient 能解密事件金额 |
| disclose / public reveal | public decrypt 或项目 callback helper | proof/签名编码符合实现 |
| wrapper | 项目 wrapper helper | 只用于 wrapper 测试准备余额 |

## 基本 helper

```ts
async function encryptAmount(tokenAddress: string, signer: HardhatEthersSigner, amount: bigint) {
  return fhevm
    .createEncryptedInput(tokenAddress, signer.address)
    .add64(amount)
    .encrypt();
}

async function decryptBalance(
  token: MyConfidentialToken,
  tokenAddress: string,
  account: string,
  signer: HardhatEthersSigner,
) {
  const handle = await token.confidentialBalanceOf(account);
  return fhevm.userDecryptEuint(FhevmType.euint64, handle, tokenAddress, signer);
}
```

## transfer

```ts
const encrypted = await encryptAmount(tokenAddress, alice, 100n);

await token
  .connect(alice)
  ["confidentialTransfer(address,bytes32,bytes)"](
    bob.address,
    encrypted.handles[0],
    encrypted.inputProof,
  );

expect(await decryptBalance(token, tokenAddress, bob.address, bob)).to.eq(100n);
```

错误路径：

- `createEncryptedInput(otherToken, alice.address)` 后调用当前 token，应失败。
- `createEncryptedInput(token, bob.address)` 后由 Alice 调用，应失败。
- 余额不足时按实现预期 revert 或 fail-closed，不能让余额变成 wrapping 结果。

## operator

operator 作为 caller 时，encrypted input 的 user 应是 operator：

```ts
await token.connect(alice).setOperator(operator.address, expiry);

const encrypted = await encryptAmount(tokenAddress, operator, 50n);
await token
  .connect(operator)
  ["confidentialTransferFrom(address,address,bytes32,bytes)"](
    alice.address,
    bob.address,
    encrypted.handles[0],
    encrypted.inputProof,
  );
```

同时测未授权和过期：

```ts
await expect(token.connect(operator).confidentialTransferFrom(...)).to.be.reverted;
```

## event amount

如果事件里暴露 encrypted amount handle：

```ts
const tx = await token.connect(alice)["confidentialTransfer(address,bytes32,bytes)"](...);
const receipt = await tx.wait();
const eventAmountHandle = extractAmountHandle(receipt!.logs);

expect(await fhevm.userDecryptEuint(FhevmType.euint64, eventAmountHandle, tokenAddress, alice)).to.eq(100n);
expect(await fhevm.userDecryptEuint(FhevmType.euint64, eventAmountHandle, tokenAddress, bob)).to.eq(100n);
```

第三方解密应失败。

## disclose

如果 token 支持公开 reveal 某个 amount：

- 测只有有权用户能 request。
- 测未 request 不能 finalize。
- 测错 handle、错 cleartext、重复 finalize 失败。
- 如果需要链上 `FHE.checkSignatures`，按项目 callback helper 生成签名/proof，不要把 user decrypt 当 public proof。

普通 token 测试不需要 wrapper helper；只有 `IERC7984ERC20Wrapper` 测试才准备 underlying 并 wrap。
